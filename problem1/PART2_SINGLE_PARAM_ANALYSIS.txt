PART 2: SINGLE PARAMETER SWEEP ANALYSIS
Comprehensive Cache Parameter Impact Study

Base Configuration:
  - L1I: 16kB, 4-way associative
  - L1D: 16kB, 4-way associative (varies for L1D sweep)
  - L2: 256kB, 8-way associative (varies for L2 sweep)
Benchmark: Matrix Multiplication (128×128, integer matrices)

# L1D CACHE SIZE ANALYSIS (16kB, 32kB, 64kB)

Observed Trends:
L1D size shows modest impact with hit rates improving from ~50.03% (16kB) to 
~50.04% (64kB), while execution time remains relatively constant around 1,974B 
ticks. The minimal variation occurs because the 192KB working set far exceeds 
all L1D sizes, causing persistent capacity misses regardless of size.

Why This Effect:
L1D thrashing dominates due to the matrix multiplication's large memory 
footprint. Even 64kB L1D can only hold one-third of a single matrix, forcing 
continuous cache line evictions. The column-major access pattern of Matrix B 
exacerbates this, as each access brings a cache line but uses only one element.

Performance Saturation:
Saturation occurs immediately at 16kB - further increases yield <0.01% hit rate 
improvement and negligible execution time reduction. L1D size is fundamentally 
limited by working set constraints.

# L2 CACHE SIZE ANALYSIS (128kB, 256kB, 512kB, 1MB)

Observed Trends:
L2 size exhibits DRAMATIC non-linear impact. Execution time plummets from 
1,974B ticks (128kB) to 750B ticks (512kB) - a 62% reduction. L2 hit rates 
explode from ~0.01% to ~95.4% at 512kB, reaching 99.7% at 1MB.

Why This Effect:
The 192KB working set creates a sharp performance cliff. Below 512kB, the L2 
cannot contain even two matrices, causing catastrophic thrashing. At 512kB+, 
temporal locality is captured, dramatically reducing main memory traffic from 
~50% to ~2.5% of all requests.

Performance Saturation:
Saturation begins at 512kB (90% of peak performance). The 512kB→1MB transition 
yields only 9% additional improvement despite doubling capacity. Beyond 1MB, 
returns diminish to <5%.

# L1 ASSOCIATIVITY ANALYSIS (2-way, 4-way, 8-way)

Observed Trends:
L1 associativity shows minimal impact with execution time varying <3% across 
all values and hit rates remaining constant at ~50.03%. The 2→4-way transition 
provides slight benefit (~1-2%), while 4→8-way is negligible.

Why This Effect:
Capacity misses dominate over conflict misses in L1D due to working set size. 
Associativity helps reduce conflict misses but cannot address fundamental 
capacity constraints. The streaming access pattern creates limited reuse 
regardless of associativity.

Performance Saturation:
Immediate saturation at 4-way associativity. Higher values provide <1% benefit, 
indicating conflict misses are already minimized and capacity dominates.

# L2 ASSOCIATIVITY ANALYSIS (4-way, 8-way, 16-way)

Observed Trends:
L2 associativity demonstrates modest impact with 4→8-way providing ~2% 
improvement and 8→16-way adding <1%. Hit rate improvements are similarly minor 
(~0.1-0.2% increase per doubling).

Why This Effect:
Once L2 capacity is sufficient, conflict misses become secondary. The matrix 
access pattern creates some interference, which associativity partially 
mitigates, but capacity misses from working set pressure dominate overall 
behavior.

Performance Saturation:
Saturation at 8-way associativity. Beyond this, diminishing returns make 
16-way cost-ineffective for this workload.

# COMPARATIVE INSIGHTS

Parameter Impact Ranking (highest to lowest):
1. L2 SIZE: 62% performance variation (dominant factor)
2. L2 ASSOCIATIVITY: ~2% variation (minor impact)
3. L1 ASSOCIATIVITY: ~2% variation (minor impact)  
4. L1D SIZE: <1% variation (negligible impact)

Critical Insight: CAPACITY dominates ASSOCIATIVITY for memory-intensive 
workloads. L2 size determines whether the working set fits in cache hierarchy, 
creating order-of-magnitude performance differences. Associativity provides 
incremental refinement but cannot overcome capacity limitations.

Design Guideline: For similar workloads, prioritize cache CAPACITY over 
associativity when allocating area budget. The optimal strategy invests heavily 
in L2 size (512kB-1MB) while accepting modest associativity (4-8 way).
